# 产品需求文档（PRD）
## AI 智能对话平台

**版本**：v1.0
**日期**：2026-03-26
**状态**：草稿

---

## 1. 产品概述

### 1.1 产品定位

AI 智能对话平台是一款基于大语言模型（LLM）的多轮对话 Web 应用，集成 DeepSeek AI 接口，提供实时流式对话、文件分析、对话历史管理等功能，面向有智能问答和文档分析需求的个人用户。

### 1.2 产品目标

- 提供流畅、低延迟的 AI 对话体验（SSE 流式输出）
- 支持文件上传，让用户借助 AI 分析文档内容
- 保存用户对话历史，支持随时回顾和继续
- 提供安全的用户账户体系（注册、登录、JWT 认证）

### 1.3 目标用户

| 用户群体 | 核心需求 |
|---------|---------|
| 开发人员 | 代码审查、技术问答、调试帮助 |
| 知识工作者 | 文档分析、报告撰写辅助 |
| 学生 | 学习辅导、作业解析 |
| 普通用户 | 日常智能问答 |

## 2. 功能需求

### 2.1 用户认证模块

#### FR-001 用户注册

- **描述**：新用户通过用户名、邮箱、密码创建账户
- **输入**：用户名（唯一）、邮箱（唯一）、密码
- **验证规则**：
  - 用户名不可与已有账户重复
  - 邮箱不可与已有账户重复
  - 密码后端 BCrypt 加密存储
- **成功响应**：返回 JWT Token，自动进入聊天界面
- **失败场景**：用户名/邮箱已存在，显示错误提示

#### FR-002 用户登录

- **描述**：已注册用户通过用户名和密码登录
- **输入**：用户名、密码
- **成功响应**：返回 JWT Token，进入聊天界面，加载历史对话列表
- **失败场景**：用户名或密码错误，显示错误提示
- **Token 有效期**：24 小时

#### FR-003 用户登出

- **描述**：清除本地 JWT Token，返回登录页
- **行为**：清空 localStorage 中的 token，清空当前会话状态

---

### 2.2 对话管理模块

#### FR-004 新建对话

- **描述**：用户发送第一条消息时自动创建新对话
- **规则**：对话标题取用户首条消息的前 50 个字符
- **存储**：对话元数据（id、title、创建时间、更新时间）持久化到数据库

#### FR-005 对话列表

- **描述**：侧边栏展示当前用户的所有历史对话
- **排序**：按最近更新时间降序排列
- **显示内容**：对话标题、相对时间
- **行为**：点击对话项加载该对话的历史消息

#### FR-006 删除对话

- **描述**：用户可删除某一对话及其所有消息
- **确认**：无需二次确认，直接删除
- **级联**：对话下的所有消息同步删除
- **响应**：删除后从列表移除，若当前对话被删除则清空聊天区

#### FR-007 加载历史消息

- **描述**：切换到某对话时，从服务端加载该对话的全部历史消息
- **排序**：消息按创建时间升序显示

---

### 2.3 聊天交互模块

#### FR-008 发送消息

- **描述**：用户在输入框输入文本，点击发送或按 Enter 键提交
- **限制**：对话进行中（流式响应未完成）时禁止再次发送
- **规则**：消息内容为空时不可发送

#### FR-009 流式 AI 响应

- **描述**：AI 回复通过 SSE（Server-Sent Events）实时推送，逐字显示
- **体验**：用户看到 AI 逐步输出文字，有"打字效果"
- **状态指示**：AI 输出期间显示打字动画指示器
- **超时**：SSE 连接超时时间 3 分钟

#### FR-010 停止响应

- **描述**：用户在 AI 流式输出期间可点击"停止"按钮终止输出
- **行为**：关闭 SSE 连接，保留已输出的部分内容

#### FR-011 重新生成

- **描述**：用户可对最后一条 AI 回复点击"重新生成"
- **行为**：保持最后一条用户消息不变，重新向 AI 发起请求
- **存储**：新的 AI 回复替换上一条 AI 消息

#### FR-012 Markdown 渲染

- **描述**：AI 回复支持 Markdown 格式渲染
- **支持格式**：标题、粗体、斜体、代码块、列表、链接等

---

### 2.4 文件上传模块

#### FR-013 上传文件

- **描述**：用户在发送消息前可上传文本类文件作为上下文
- **支持格式**：`.txt, .md, .json, .csv, .xml, .js, .ts, .jsx, .tsx, .java, .py, .html, .css, .yaml, .yml, .sh, .log`
- **大小限制**：单文件最大 10MB
- **行为**：上传成功后文件内容作为附件追加到用户消息中发送给 AI

#### FR-014 附件管理

- **描述**：已上传的文件显示为附件标签
- **操作**：用户可移除已上传的附件
- **发送后**：消息发送完成后自动清空附件列表

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 首条 AI Token 响应延迟 | ≤ 3 秒 |
| 登录/注册接口响应时间 | ≤ 1 秒 |
| 对话列表加载时间 | ≤ 2 秒 |
| 文件上传（10MB 内） | ≤ 5 秒 |
| SSE 连接最大时长 | 3 分钟 |

### 3.2 安全需求

- 用户密码使用 BCrypt 加密存储，不可逆
- 所有需认证接口通过 JWT Bearer Token 验证
- Token 有效期 24 小时，过期后需重新登录
- CORS 仅允许指定来源访问后端接口
- 用户数据严格隔离，用户只能访问自己的对话和消息

### 3.3 可用性需求

- 支持主流浏览器（Chrome、Firefox、Edge 最新版）
- 响应式布局，支持主流桌面分辨率
- 网络异常时显示友好的错误提示

### 3.4 兼容性需求

- 前端：Node.js 18+，支持 ES2020+
- 后端：Java 17+，Spring Boot 3.2.5
- 数据库：MySQL 8.0+

## 4. 业务规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-001 | 每个用户的对话相互隔离，不可访问他人对话 |
| BR-002 | AI 上下文最多携带最近 40 条消息 |
| BR-003 | 对话标题由首条用户消息的前 50 个字符自动生成 |
| BR-004 | 消息一旦发送不可编辑，仅支持重新生成 AI 回复 |
| BR-005 | 文件内容以文本形式追加到用户消息末尾提交给 AI |
| BR-006 | 重新生成时不重复写入用户消息，只替换 AI 回复 |

## 5. 用户界面需求

### 5.1 整体布局

```
┌─────────────────────────────────────────────────┐
│  侧边栏（对话列表）  │     主聊天区域             │
│                    │                             │
│  [+ 新对话]         │  消息列表                  │
│  ─────────────     │  ┌─────────────────────┐   │
│  对话 1            │  │ 用户消息             │   │
│  对话 2            │  └─────────────────────┘   │
│  对话 3            │  ┌─────────────────────┐   │
│  ...              │  │ AI 回复（Markdown）  │   │
│                   │  └─────────────────────┘   │
│  ─────────────    │                             │
│  [登出]           │  [输入框] [文件] [发送]      │
└────────────────────────────────────────────────┘
```

### 5.2 认证界面

- 未登录时全页显示登录/注册表单
- 支持登录和注册表单切换
- 表单提交时显示 loading 状态
- 错误信息实时显示

### 5.3 聊天界面

- 用户消息靠右显示，AI 消息靠左显示
- AI 输出中显示打字动画
- 消息列表自动滚动到最新消息
- 文件附件以标签形式显示在输入框上方

---

## 6. 数据需求

### 6.1 数据存储

- 用户账户信息持久化存储（MySQL）
- 所有对话和消息持久化存储（MySQL）
- JWT Token 存储于客户端 localStorage

### 6.2 数据保留

- 用户主动删除前，对话和消息永久保留
- 用户注销账户目前不支持（v1.0 范围外）

---

## 7. 约束和假设

### 7.1 约束

- 依赖 DeepSeek API 服务可用性
- AI 响应质量依赖 DeepSeek 模型能力
- 单文件上传限制 10MB（文本类文件）

### 7.2 假设

- 用户使用现代浏览器访问
- 服务器与 DeepSeek API 网络连通
- 初期用户量较小，单机部署即可满足需求
